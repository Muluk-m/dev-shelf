  <name>Task 1: Create export database function and API endpoint</name>
  <files>lib/database/export.ts, workers/routes/export.ts, workers/app.ts</files>
  <action>
1. Create `lib/database/export.ts` with an `exportAllData(db: D1Database)` function:
   - Define `ExportData` interface with `version: string`, `exportedAt: string`, and `data` containing `tools: Tool[]` and `categories: ToolCategory[]`.
   - Query all four tables in parallel using `Promise.all` with `db.prepare().all()` -- do NOT use the cached `getTools()`/`getToolCategories()` functions. Export must always return fresh data.
   - Map tools from snake_case to camelCase matching the existing `Tool` interface: `is_internal` -> `isInternal` (Boolean), `last_updated` -> `lastUpdated`, `permission_id` -> `permissionId`.
   - For each tool, filter `tool_environments` by `tool_id` and map `is_external` -> `isExternal` (Boolean). Include only `name`, `label`, `url`, `isExternal` fields.
   - For each tool, filter `tool_tags` by `tool_id` and extract the `tag` string values into a `tags: string[]` array.
   - Map categories: include `id`, `name`, `description`, `icon`, `color` (no snake_case mapping needed for categories).
   - Return the assembled `ExportData` object with `version: "1.0"` and `exportedAt: new Date().toISOString()`.

2. Create `workers/routes/export.ts` with a Hono router:
   - Follow the exact pattern from `workers/routes/tools.ts`: `new Hono<{ Bindings: Cloudflare.Env }>()`.
   - Single GET `/` handler that:
     a. Calls `exportAllData(c.env.DB)`.
     b. Serializes to JSON with `JSON.stringify(exportData, null, 2)`.
     c. Returns a `new Response(json, { headers })` with:
        - `Content-Type: application/json`
        - `Content-Disposition: attachment; filename="devhub-export-YYYY-MM-DD.json"` (use current date).
     d. Wraps in try/catch, returns `c.json({ error: "Export failed" }, 500)` on error.
   - Export the router as `exportRouter`.

3. Mount the export route in `workers/app.ts`:
   - Import `exportRouter` from `./routes/export`.
   - Add `app.route("/api/export", exportRouter)` alongside the existing API routes (after the categories route).

Note on auth: The existing `authMiddleware` in this codebase allows GET requests on `/api/*` paths without authentication. After Phase 3 implements RBAC, the export endpoint should be restricted to admin role. For now, add a comment `// TODO: restrict to admin role after Phase 3 RBAC` in the route handler. The current middleware will still require auth for non-GET requests, and GET is appropriate for export.
