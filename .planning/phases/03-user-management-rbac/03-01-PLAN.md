  <action>
  1. **Create `workers/middleware/rbac.ts`:**
     - Export `requireAdmin` middleware using `createMiddleware` from `hono/factory`
     - Extract user from Hono context (set by Phase 2's auth middleware as `c.get('user')`)
     - If no user or `user.role !== 'admin'`, return `c.json({ error: 'Forbidden: Admin access required' }, 403)`
     - Otherwise call `next()`
     - Export `requireAuth` middleware that checks if any user is set in context (401 if not)

  2. **Create `lib/database/users.ts`:**
     - Define `User` interface: `{ id, username, displayName, role, passwordHash, salt, createdAt, updatedAt }`
     - Define `UserPublic` interface (without passwordHash/salt): `{ id, username, displayName, role, createdAt }`
     - Implement `getUserCount(db: D1Database): Promise<number>` - SELECT COUNT(*) FROM users
     - Implement `getUserById(db: D1Database, id: string): Promise<User | null>` - with snake_case to camelCase mapping
     - Implement `getUserByUsername(db: D1Database, username: string): Promise<User | null>`
     - Implement `getAllUsers(db: D1Database): Promise<UserPublic[]>` - Returns all users WITHOUT password data
     - Implement `updateUserPassword(db: D1Database, userId: string, passwordHash: string, salt: string): Promise<void>`
     - Implement `updateUserRole(db: D1Database, userId: string, role: 'admin' | 'user'): Promise<void>`
     - Follow existing `lib/database/tools.ts` patterns for snake_case to camelCase field mapping
     - Use `@cloudflare/workers-types` for D1Database type import

  3. **Create `workers/routes/setup.ts`:**
     - Create Hono router with `Cloudflare.Env` bindings
     - `GET /status` endpoint:
       - Call `getUserCount(c.env.DB)` from lib/database/users.ts
       - Return `{ initialized: count > 0, needsSetup: count === 0 }`
       - This endpoint must be PUBLIC (no auth required)
     - `POST /init` endpoint:
       - First check `getUserCount` - if > 0, return `{ error: 'System already initialized' }` with 400
       - Parse body: `{ username: string, password: string, displayName?: string }`
       - Validate: username min 3 chars, password min 8 chars
       - Generate UUID for user id, random salt
       - Hash password using Phase 2's `hashPassword` utility (import from workers/utils/auth or equivalent)
       - INSERT into users table with `role = 'admin'`
       - Create JWT with `{ sub: id, username, role: 'admin' }` using Phase 2's `createJWT`
       - Set auth_token cookie (httpOnly, secure, sameSite: lax, 30 day maxAge)
       - Return `{ message: 'Admin account created', user: { id, username, displayName, role } }`
       - This endpoint must be PUBLIC (no auth required) but self-guarding (rejects if users exist)
     - Export as `setupRouter`

  4. **Create `workers/routes/admin.ts`:**
     - Create Hono router with `Cloudflare.Env` bindings
     - Import `requireAdmin` from middleware/rbac.ts
     - `GET /users` endpoint (requireAdmin):
       - Call `getAllUsers(c.env.DB)` from lib/database/users.ts
       - Return `{ users: UserPublic[] }`
     - `POST /users/:id/reset-password` endpoint (requireAdmin):
       - Parse body: `{ newPassword: string }`
       - Validate: newPassword min 8 chars
       - Verify target user exists via getUserById
       - Generate new salt, hash password using Phase 2's hashPassword
       - Call updateUserPassword
       - Return `{ message: 'Password reset successfully' }`
     - `PUT /users/:id/role` endpoint (requireAdmin):
       - Parse body: `{ role: 'admin' | 'user' }`
       - Validate role value
       - Prevent admin from demoting themselves (compare c.get('user').id with param id)
       - Call updateUserRole
       - Return `{ message: 'Role updated successfully' }`
     - Export as `adminRouter`

  5. **Create `db/migrations/0003_add_role_column.sql`:**
     - This migration is a safety net in case Phase 2 doesn't include the role column
     - Use `ALTER TABLE users ADD COLUMN role TEXT NOT NULL DEFAULT 'user' CHECK (role IN ('admin', 'user'))` wrapped in a check
     - Since D1 SQLite doesn't support IF NOT EXISTS for ALTER TABLE, use a try-catch pattern or check column existence
     - Alternative: If Phase 2 already includes role column, this file can be a no-op comment explaining that

  NOTE: The exact import paths for Phase 2's auth utilities (hashPassword, createJWT) depend on Phase 2's implementation. Use `../../workers/utils/auth` or `../../lib/auth` based on what Phase 2 creates. If the imports don't resolve, check Phase 2's SUMMARY.md for actual file locations.
